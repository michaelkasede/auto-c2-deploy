Rule: Apply when editing .yml or .yaml files in the playbooks/ or roles/ directory.

- Use fully qualified collection names (FQCN) like ansible.builtin.apt.
- Every task must have a descriptive name:.
- Use become: true only when necessary for package installation.
- Assume the inventory is dynamic and generated by Terraform; do not hardcode IP addresses.
- Use fully qualified collection names (e.g., `ansible.builtin.copy`).
- Ensure all playbooks are idempotent (running twice changes nothing).
- Use dynamic inventories where possible.
